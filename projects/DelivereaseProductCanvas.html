DeliverEase

A comparison-first food delivery app — systems analysis and product canvas from scratch

Concept

DeliverEase is a concept mobile app that helps users compare delivery fees, ETA, and promotions across multiple food delivery platforms in one place — reducing decision fatigue and app-switching while making ordering faster and more cost-effective.

Course: Systems Analysis & Design  |  Focus: Requirements → User Flow → Mockups  |  Deliverables: Personas, Epic, User Stories, Workflow, UI Mockups

Problem

  • App-switching tax: users manually compare platforms, which is slow and frustrating.
  • Unclear total cost: fees and promotions vary by platform and change frequently.
  • Decision fatigue: too many options, too little clarity at the point of decision.

Solution

  • Search a restaurant or dish once (with fuzzy search support).
  • See a side-by-side comparison across platforms: delivery fees, ETA, active promotions.
  • Pick the best option, then hand off to the selected platform for checkout.

My Role

  • Translated the concept into system requirements (epic, user stories with acceptance criteria).
  • Created user personas and mapped the end-to-end user journey.
  • Designed workflow, storyboard, and UI mockups to communicate intended behavior clearly.
  • Iterated on feasibility improvements to keep the concept realistic without diluting the core value.

Example User Stories

  • Search: As a user, I want to search restaurants or menu items so I can quickly find what I'm craving. Results populate within 2 seconds; supports partial keywords.
  • Fuzzy matching: As a user, I want suggestions even if my search term isn't exact. Top suggestions appear for typos; up to 10 relevant results shown.
  • Comparison: As a user, I want to compare delivery fees, ETA, and promotions across platforms. Comparison loads within 3 seconds; sorting updates within 1 second.

System Architecture

  • Mobile app: Search, compare, redirect (iOS/Android)
  • Backend API: Auth, aggregation, caching
  • Data store: User preferences, search history, redirect logs, metrics
  • 3rd-party sources: Official delivery platform APIs; public pricing/ETA data

Highlights

  • Measurable requirements: concrete response time targets for search and sorting.
  • Clear value proposition: faster decisions with potential cost savings quantified.
  • Full product thinking: competitive landscape analysis and freemium revenue model included.
